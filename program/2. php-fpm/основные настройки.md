***Лимит открытых файлов*** ^f703bb

> [!NOTE] Title
> PHP message: PHP Warning:  include(/srv/www/html/---/back/src/Models/---/---.php): ***Failed to open stream: Too many open files***

По умолчанию для php настроен лимит открытых файлов в количестве равным 1024

 1. Создайте файл переопределения лимитов для сервиса:
 ```bash
 sudo systemctl edit php-fpm.service
 ```
 2. В открывшемся окне вставьте:
   ```
   [Service]
   LimitNOFILE=65535
   ```
 3. Откройте конфиг вашего пула
```bash
sudo vim /etc/php8/fpm/php-fpm.d/www.conf
```
 4. Найдите параметр rlimit_files и раскомментируйте его (или добавьте):
```
rlimit_files = 65535
```
 5. Примените изменения:
 ```bash
 sudo systemctl daemon-reload
 ```
 ```bash
 sudo systemctl reload php-fpm.service
 ```
 6. Проверка - Смотрим лимиты:
```bash
sudo systemctl status php-fpm.service | grep PID: | cut -d ' ' -f6 | xargs -I {} cat /proc/{}/limits | grep 'Max open files'
```

****Размер файлов*** ^2019a0
 
  *В PHP нужно изменить два обязательных параметра и один желательный.*

   * **Где искать:**
      Путь зависит от версии PHP. Обычно это:
       * /etc/php/8.x/fpm/php.ini (замените 8.x на вашу версию, например 8.2)
   * **Что изменить:**
       1. upload_max_filesize — максимальный размер одного файла.
       2. post_max_size — максимальный размер всех данных, отправляемых методом POST (должен быть больше или равен upload_max_filesize).
       3. memory_limit — (желательно) сколько памяти может потреблять скрипт. Должен быть больше, чем post_max_size.

> [!NOTE] Пример параметров:
> 	   upload_max_filesize = 100M
> 	   post_max_size = 105M
> 	   memory_limit = 256M

***Важная цепочка лимитов:***

  Чтобы загрузка файла размером 100М прошла успешно, соблюдайте иерархию:
  Nginx client_max_body_size (110M) > PHP post_max_size (105M) > PHP upload_max_filesize (100M).

  Если Nginx лимит будет меньше, вы получите ошибку 413 Request Entity Too Large.
  Если PHP лимит будет меньше, файл просто не попадет в массив $_FILES.
