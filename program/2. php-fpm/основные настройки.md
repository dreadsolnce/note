***Лимит открытых файлов*** ^f703bb

> [!NOTE] Title
> PHP message: PHP Warning:  include(/srv/www/html/---/back/src/Models/---/---.php): ***Failed to open stream: Too many open files***

По умолчанию для php настроен лимит открытых файлов в количестве равным 1024

 1. Создайте файл переопределения лимитов для сервиса:
 ```bash
 sudo systemctl edit php-fpm.service
 ```
 2. В открывшемся окне вставьте:
   ```
   [Service]
   LimitNOFILE=65535
   ```
 3. Откройте конфиг вашего пула
```bash
sudo vim /etc/php8/fpm/php-fpm.d/www.conf
```
 4. Найдите параметр rlimit_files и раскомментируйте его (или добавьте):
```
rlimit_files = 65535
```
 5. Примените изменения:
 ```bash
 sudo systemctl daemon-reload
 ```
 ```bash
 sudo systemctl reload php-fpm.service
 ```
 6. Проверка - Смотрим лимиты:
```bash
sudo systemctl status php-fpm.service | grep PID: | cut -d ' ' -f6 | xargs -I {} cat /proc/{}/limits | grep 'Max open files'
```
