
### Режим PrivateTmp

> [!NOTE] Описание
> Работа встроенного Garbage Collector (GC) в режиме PrivateTmp имеет одну критическую особенность: 
>  - вы **обязаны** включить его внутри PHP, так как внешние (системные) инструменты очистки такие как `phpsessionsclean.service` больше не имеют доступа к вашим файлам.
> 
> Это единственный способ обработки устаревших файлов сессий

1. **Почему в режиме PrivateTmp не работает обычная очистка?**
  Когда в systemd включено PrivateTmp=yes:
   * Служба PHP-FPM видит свою изолированную папку /tmp.
   * Системный скрипт /usr/lib/php/sessionclean запускается «снаружи» в общем пространстве имен. Он ищет сессии в основной папке /tmp или /var/lib/php/sessions.
   * Результат: Системный скрипт не видит файлов внутри systemd-private-..., и они копятся там вечно, пока вы не перезагрузите службу.

2. **Как настроить встроенный GC (Единственный рабочий способ)**

  Чтобы PHP сам чистил свои сессии внутри изоляции, вам нужно изменить настройки в конфиге /etc/php/8.4/fpm/php.ini или конфиге пула (например, /etc/php/8.4/fpm/pool.d/www.conf). 

  Добавьте эти строки:

- Указываем PHP, где искать файлы сессий внутри его изоляции
`php_value[session.save_path] = "/tmp"`

- Включаем встроенный Garbage Collector
 ; В Debian/Ubuntu он по умолчанию 0 (выключен), его НУЖНО включить
`php_value[session.gc_probability] = 1`
 
- Устанавливаем делитель (вероятность 1/100)
`php_value[session.gc_divisor] = 100`
 
- Устанавливаем время жизни сессии (например, 24 минуты)
`php_value[session.gc_maxlifetime] = 1440`

3. **Как это работает пошагово внутри PrivateTmp**

 - Запрос: Пользователь заходит на сайт, срабатывает session_start().
 - Вероятность: PHP «подбрасывает кубик». С вероятностью 1% (из настроек выше) он решает запустить GC.
 - Сканирование: PHP смотрит внутрь /tmp. Благодаря PrivateTmp, он видит только свои сессии. Это даже ускоряет процесс, так как ему не нужно просматривать чужие временные файлы.
 - Сравнение: PHP берет каждый файл sess_* и проверяет время его последнего изменения.
 - Удаление: Если файл старше 1440 секунд, PHP сам (от имени пользователя www-data) удаляет его.
 - Завершение: Процесс продолжается, пользователь получает свою страницу.
