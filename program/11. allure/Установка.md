✦ Развертывание Allure TestOps на Linux-сервере включает в себя несколько шагов. Ниже приведено пошаговое руководство для развертывания Allure TestOps на сервере под управлением Linux (например, Ubuntu).

    ---

    Подготовка сервера

    1. Обновление системы
    Сначала убедитесь, что ваш сервер обновлен:

     1 sudo apt update && sudo apt upgrade -y

    2. Установка Docker и Docker Compose
    Allure TestOps можно развернуть с использованием Docker, поэтому установите Docker и Docker Compose:

      1 # Установка Docker
      2 sudo apt install apt-transport-https ca-certificates curl software-properties-common -y
      3 curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
      4 sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
      5 sudo apt update
      6 sudo apt install docker-ce -y
      7 
      8 # Установка Docker Compose
      9 sudo curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
     10 sudo chmod +x /usr/local/bin/docker-compose

    3. Создание директории для Allure TestOps
    Создайте директорию, где будут храниться конфигурационные файлы и данные:

     1 mkdir ~/allure-testops && cd ~/allure-testops

    ---

    Развертывание Allure TestOps

    4. Создание docker-compose.yml файла
    Создайте файл docker-compose.yml со следующим содержимым:

      1 version: '3.7'
      2 
      3 services:
      4   postgres:
      5     image: postgres:13
      6     container_name: allure-postgres
      7     environment:
      8       POSTGRES_USER: allure
      9       POSTGRES_PASSWORD: allure
     10       POSTGRES_DB: allure
     11     volumes:
     12       - postgres_data:/var/lib/postgresql/data
     13     ports:
     14       - "5432:5432"
     15 
     16   allure-testops:
     17     image: allure/testops:latest
     18     container_name: allure-testops
     19     depends_on:
     20       - postgres
     21     ports:
     22       - "8080:8080"
     23     environment:
     24       DB_HOST: postgres
     25       DB_PORT: 5432
     26       DB_USER: allure
     27       DB_PASSWORD: allure
     28       DB_NAME: allure
     29     volumes:
     30       - allure_data:/opt/allure/data
     31 
     32 volumes:
     33   postgres_data:
     34   allure_data:

    5. Запуск контейнеров
    Теперь запустите Allure TestOps и PostgreSQL с помощью Docker Compose:

     1 docker-compose up -d

    6. Проверка состояния контейнеров
    Вы можете проверить состояние контейнеров с помощью команды:

     1 docker-compose ps

    После запуска контейнеров сервис Allure TestOps будет доступен по адресу:

     1 http://<ваш-IP>:8080

    ---

    Настройка Allure TestOps

     7. Первый вход
       Откройте браузер и перейдите по адресу http://<ваш-IP>:8080. Вы увидите экран настройки. Следуйте инструкциям для завершения первоначальной настройки.

     8. Создание пользователя
       Создайте учетную запись администратора и войдите в систему.

     9. Настройка интеграций
       Вы можете настроить интеграции с системами контроля версий (например, GitHub, GitLab), CI/CD-сервисами и т. д.

    ---

    Заключение
    Вы успешно развернули Allure TestOps на Linux-сервере с использованием Docker. Теперь вы можете использовать его для управления тестами, отслеживания результатов и интеграции с вашими инструментами CI/CD.
