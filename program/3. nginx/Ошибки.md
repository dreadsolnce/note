#### 1. "/usr//off" failed (30: Read-only file system)

![[Снимок экрана от 2026-01-16 14-37-09.png]]

> [!warning] Причина
> **Ошибка в директиве error_log**  
> Директиву error_log ***нельзя*** просто выключить словом off.
> error_log off; — это заставит Nginx пытаться создать файл с именем off в корневой директории (которая в Docker или защищенных системах часто Read-only).


> [!NOTE] Решение
> Чтобы «выключить» логи ошибок, направьте их в «черную дыру»:
> ```
> error_log /dev/null crit;
> ```
