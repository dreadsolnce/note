> [!NOTE] Title
> директива **server_name _** (подчеркивание) означает создание «catch-all» (захват всего) сервера по умолчанию. Это указывает Nginx обрабатывать запросы, которые не соответствуют ни одному другому явно заданному server_name на данном IP-адресе и порту, часто используется для заглушек или сброса неверных запросов

**Ключевые особенности `server_name _`:**

- **Действие:** Это не какое-то специальное имя, а просто любая строка, которая не совпадает с именами сайтов (например, зарезервированный символ), поэтому часто используют `_`, `default` или даже `""`.
- **Приоритет:** Обычно используется в комбинации с `listen 80 default_server;`, чтобы гарантировать, что этот блок — первый кандидат при отсутствии точного совпадения.
- **Применение:** Защита от парковки доменов, предотвращение показа случайных сайтов при прямом обращении к IP-адресу, обработка запросов с неверным заголовком `Host`

==**Пример использования:**==

```
server {
    listen 80 default_server;
    server_name _; # Захватывает все запросы, не попавшие в другие server_name
    return 444;    # Обрывает соединение, ничего не возвращая
}
```
