#### Пример настройки запуска своего скрипта

1. **Создаем скрипт sh и размещаем его на клиенте в папке /etc/zabbix/scripts**

Изменяем владельца и делаем его исполняемым:

```
sudo chown zabbix:zabbix /etc/zabbix/scripts/domain_ssl_check.sh && sudo chmod zabbix:zabbix /etc/zabbix/scripts/domain_ssl_check.sh

```

![[Снимок экрана от 2026-01-28 10-22-15.png]]

2. **В конфигурационном файле zabbix_agent2.conf прописываем ключ `AllowKey`**


> [!NOTE]
> С помощью данного параметра мы разрешим запуск команды выполнения скрипта на клиенте, т.к. по умолчанию запуск стронних скриптов запрещен.

Просмотр параметров конфигурационного файла 

```
cat /etc/zabbix/zabbix_agent2.conf | grep -vP "^\s*$" | grep -v '#'
```

![[Снимок экрана от 2026-01-28 10-30-59.png]]

```
sudo vim /etc/zabbix/zabbix_agent2.conf
```

	+++ AllowKey=system.run[/etc/zabbix/scripts/domain_ssl_check.sh*]


![[Снимок экрана от 2026-01-28 10-37-20.png]]

Перезапускаем службу zabbix:

```
sudo systemctl restart zabbix-agent2.service
```
