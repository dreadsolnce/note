✦ Для того чтобы процессы PM2 автоматически запускались после перезагрузки сервера, необходимо выполнить процедуру генерации и сохранения скрипта автозапуска (startup script).

- ***Сгенерируйте команду автозапуска***
*Выполните команду:*
```bash
pm2 startup
```
  
> [!NOTE] Title
> ***Важно:*** В ответ PM2 выведет в терминал длинную команду, которую вам нужно скопировать и выполнить вручную. Она выглядит примерно так:
> sudo env PATH=$PATH:/home/user/bin /usr/lib/node_modules/pm2/bin/pm2 startup systemd -u user --hp /home/user

===Эта команда регистрирует PM2 в системном менеджере (обычно это systemd) как службу, которая будет стартовать при загрузке ОС.===

- ***Сохраните текущий список процессов***
*После того как вы выполнили команду из предыдущего шага, нужно «заморозить» текущий список запущенных процессов:*
```bash
pm2 save
```

===Эта команда сохраняет текущее состояние (какие приложения запущены, их пути, флаги --watch и т.д.) в специальный файл ~/.pm2/dump.pm2. Именно этот файл PM2 считывает при загрузке системы.===

***Полезные дополнения:***
* Если вы добавили новый процесс или изменили старый:
   Просто снова выполните pm2 save. Повторно делать pm2 startup не нужно.
* Как проверить, что всё работает?
   Вы можете сымитировать перезагрузку службы PM2:

```
sudo systemctl restart pm2-<username>
```
(где username — имя вашего пользователя в системе). После этого введите pm2 list, и вы увидите, что процессы поднялись сами.

- Как отключить автозапуск?
   Если он больше не нужен, введите:
```
pm2 unstartup
```
И выполните предложенную в ответ команду.
