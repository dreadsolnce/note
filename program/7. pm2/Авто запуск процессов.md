✦ Для того чтобы процессы PM2 автоматически запускались после перезагрузки сервера, необходимо выполнить процедуру генерации и сохранения скрипта автозапуска (startup script).

```
sudo -u user -i
```

1. ***С генерируйте команду авто запуска***

*Выполните команду:*
```bash
pm2 startup
```

![[Снимок экрана от 2026-05-18 10-57-39.png]]

> [!NOTE] Title
> ***Важно:*** В ответ PM2 выведет в терминал длинную команду, которую вам нужно скопировать и выполнить вручную. Она выглядит примерно так:
> sudo env PATH=$PATH:/home/user/bin /usr/lib/node_modules/pm2/bin/pm2 startup systemd -u user --hp /home/user

2. ***Изменить шаблон команды перед выполнением***

	- изменить ключ --hp на путь к папке /opt/pm2daemon

![[Снимок экрана от 2026-05-18 11-20-49.png]]

3. ***Выполнить команду:***

```
sudo env PATH=$PATH:/home/kvl/.nvm/versions/node/v24.13.1/bin /usr/local/lib/node_modules/pm2/bin/pm2 startup systemd -u kvl --hp /opt/pm2daemon
```
*Команда от Лапушкина Дмитрия:*
```
sudo env PATH=$PATH:/usr/bin /usr/local/lib/node_modules/pm2/bin/pm2 startup systemd -u user --hp /opt/pm2daemon (автозапуск, чтоб при перезагрузке поднялось)
```


![[Снимок экрана от 2026-05-18 11-25-20.png]]

> [!NOTE] Описание
> Эта команда регистрирует PM2 в системном менеджере (обычно это systemd) как службу, которая будет стартовать при загрузке ОС.
> Создаст файл службы `/etc/systemd/system/pm2-kvl.service`

4. ***Изменить файл созданной службы pm2-kvl.service***

В файле `/etc/systemd/system/pm2-kvl.service` изменить значение двух переменных (убрать из пути скрытую папку .pm2):

- Environment=PM2_HOME=
- PIDFile=

![[Снимок экрана от 2026-05-18 11-28-39.png]]

на значения

- Environment=PM2_HOME=/opt/pm2daemon
- PIDFile=/opt/pm2daemon/pm2.pid

![[Снимок экрана от 2026-05-18 11-31-36.png]]

```
sudo systemctl edit pm2-kvl.service
```

![[Снимок экрана от 2026-05-18 11-47-51.png]]

5. ***Перечитать конфигурацию systemd***

```
sudo systemctl daemon-reload
```

6. ***Перезапустить службу pm2-kvl.sevice***

```
sudo systemctl restart pm2-kvl.service ; sudo systemctl status pm2-kvl.service
```

![[Снимок экрана от 2026-05-18 11-49-01.png]]

7. ***Сохраните текущий список процессов***

*После того как вы выполнили команду из предыдущего шага, нужно «заморозить» текущий список запущенных процессов:*

```bash
pm2 save
```


> [!NOTE] Описание
> Эта команда сохраняет текущее состояние (какие приложения запущены, их пути, флаги --watch и т.д.) в специальный файл /opt/pm2daemon/dump.pm2. Именно этот файл PM2 считывает при загрузке системы.
> pm2 save: Создает «снимок» всех запущенных в данный момент приложений. При загрузке системы PM2 обратится к этому снимку и запустит всё, что там было.


***Полезные дополнения:***
* Если вы добавили новый процесс или изменили старый:
   Просто снова выполните pm2 save. Повторно делать pm2 startup не нужно.
* Как проверить, что всё работает?
   Вы можете сымитировать перезагрузку службы PM2:

```
sudo systemctl restart pm2-<username>
```
(где username — имя вашего пользователя в системе). После этого введите pm2 list, и вы увидите, что процессы поднялись сами.

- Как отключить автозапуск?
   Если он больше не нужен, введите:
```
pm2 unstartup
```

И выполните предложенную в ответ команду.

Сбросьте состояние «failed» или «not-found»

sudo systemctl reset-failed
