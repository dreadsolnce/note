✦ ***Чтобы посмотреть Cache Hit Ratio (коэффициент попадания в кэш) в PostgreSQL, нужно выполнить один SQL-запрос.***

> [!NOTE] Описание
> Этот показатель показывает, какой процент данных база берет из оперативной памяти (`Shared Buffers`), а не читает с диска. В идеале он должен быть выше `99%`

**Самый простой запрос:**
  Скопируйте и выполните этот код в любом SQL-редакторе (pgAdmin, DBeaver, psql):

```
sudo -u postgres -i
```

```
psql -h localhost -U <userdb> -d <dbname>
```

```
SELECT sum(heap_blks_hit) / (sum(heap_blks_read) + sum(heap_blks_hit) + 0.000001) AS cache_hit_ratio FROM pg_statio_user_tables;
```

---
>  *Что значат цифры:*
> 	- 0.99 и выше (99%+) — Отлично. Почти все данные читаются из памяти.
> 	- 0.95 (95%) — Хорошо, но стоит присмотреться.
> 	- Ниже 0.90 (90%) — Плохо. Базе не хватает оперативной памяти, она постоянно «тормозит» об диск.   
___

✦ ***Если хотите посмотреть детально по каждой базе данных:***


> [!NOTE] Описание
> Иногда одна база работает быстро, а другая тормозит. Посмотреть по базам можно так:

```
SELECT datname AS database_name, 100 * blks_hit / (blks_read + blks_hit + 1) AS cache_hit_ratio_percent FROM pg_stat_database ORDER BY cache_hit_ratio_percent DESC;
```

___
>  *Как это работает:*
> 	 - blks_hit — сколько раз данные нашлись в памяти.
> 	 - blks_read — сколько раз пришлось идти на диск.
> 	 - Формула просто считает их соотношение.
___
