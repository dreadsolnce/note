==**Проверка времени работы транзакций**==

```
sudo -u postgres psql -c "
SELECT
    pid,
    backend_type,
    xact_start,
    query_start,
    now() - xact_start AS xact_age,
    now() - query_start AS query_age,
    state,
    wait_event,
    query
FROM pg_stat_activity
WHERE xact_start IS NOT NULL OR query_start IS NOT NULL
ORDER BY xact_age DESC NULLS LAST
LIMIT 5;"
```

> [!NOTE] Title
> Если в ответе появится строка с xact_age или query_age в 7 дней — мы нашли виновника.
> Обратите внимание на колонку backend_type. Иногда «стариком» оказывается не пользовательский запрос, а какой-нибудь logical replication launcher или autovacuum worker, который «завис».

**Принудительное закрытие зависшего запроса**

```
sudo -u postgres psql -c "SELECT pg_terminate_backend(15204);"
```

___

===**Показать зависшие процессы**===

```
sudo -u postgres psql -c "
SELECT pid, state, 
       now() - state_change AS duration, 
       query, 
       wait_event_type, wait_event 
FROM pg_stat_activity 
WHERE state != 'active' 
ORDER BY duration DESC;"
```

> [!NOTE] Title
> Если вы видите там сессии со state = 'idle in transaction' и длительностью более нескольких минут — у вас проблема.
