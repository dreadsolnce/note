***Основные команды для тестирования сети:***
##### nmap
```
nmpap -sT 8.8.8.8 -p 9094
```
> [!NOTE] Описание
> `filtered` - (фильтруется) в Nmap означает, что ==Nmap не получил ответа на свой запрос к порту, так как он, скорее всего, блокируется брандмауэром или другим сетевым фильтром==. Nmap не может определить, открыт порт или закрыт, из-за этой блокировки

##### traceroute
*По умолчанию traceroute работает по UDP протоколу*
```
traceroute 8.8.8.8
```

*ICMP протокол:*
```
traceroute --icmp 8.8.8.8
```
```
traceroute -I 8.8.8.8
```

*TCP протокол по порту **443**
```
traceroute --tcp -p 443 8.8.8.8
```

***Изменить mtu сетевой карты****
```
sudo ip link set <имя_интерфейса> mtu <новое_значение>
```

***Ping с заданным mtu с запретом фрагментации (какой максимальный размер пакета может пройти без его фрагментации)***
```
ping -c <колличество пакетов>-s <размер пакета> -M do <новое назначение>
```

***TCPDUMP Захват (анализ) сетевого трафика***
```
sudo tcpdump -i eth0
```
***Проверка (сканирование) портов с помощью tcpdump***
```
sudo tcpdump -i enp0s8 port 80 and tcp and src 192.168.192.1
```
***Захват пакетов  ICMP***
```
sudo tcpdump -i <интерфейс> icmp
```
***Захват всего пакета ssh вместо ограничения длины в 36 bit, т.к. tcpdump ограничивает размер захватываемых данных для экономии места***
```
tcpdump -s0 -i eth0
```
