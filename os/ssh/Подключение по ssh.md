***Подключение к удалённой машине через другую машину (jump - прыжок)***

```
ssh -J user@173.19.35.23 user@173.19.35.123
```

> [!NOTE] Описание
> Это значит примерно следующее: допустим мы с помощью terraform развернули две машины одну с публичным адресом вторую нет. При развертывании мы указали свой ssh ключ для доступа. Так при подключении к машине с публичным адресом мы подключимся, но вот ко второй не сможем, даже если сначала подключиться к машине с публичным адресом, то с нее мы не сможем подключиться к другой, т.к. на второй машине нет информации о первой машине. Вот в таких случаях нам и поможет jump, а именно мы как бы пробрасываем подключение (наш приватный ключ) с помощью машины с публичным ip адресом но заходим на вторую машину используя первую, а так как на второй машине у нас есть публичный ключ нашей локальной машины то мы подключимся по протоколу ssh

***Подключение с использованием тунеля:*** ^5fb1ca

В первом терминале выполнить подключение:

```
ssh -L 127.0.0.1:12312:10.15.61.129:22 user@10.89.1.200
```

```
ssh -L 127.0.0.1:12312:10.15.61.129:22 Interactive@ai-serv1p-10.15.61.129:SSH:user@10.89.1.200
```

Проверить порт:

```
netstat -tlpn
```

Во втором терминале использовать следующее подключение:

```
ssh user@127.0.0.1 -p 12312
```

***Проброс порта на удалённый сервер*** ^5b91cb

__Например для доступа удалённого сервера к интернету (например если на нём все заблокировано):

*Устанавливаем на локальном компьютере программу squid*

*На локальном компьютере убеждаемся, что нужный порт поднялся*

*Прокидываем локальный порт 3128 на удалённый сервер:*

```
ssh -R 3128:127.0.0.1:3128 <username - сервер>@<ipaddress - сервер>
```

Для skdpu команда выглядит вот так:

```
ssh -R 3080:127.0.0.1:3080 Interactive@ai-serv2t-10.15.61.130:SSH:kolchinvll@10.6.236.80
```

*Смотрим, что бы на удалённом сервере поднялся порт 3128*

*Делаем export http_proxy и https_proxy*

```
export https_proxy=http://127.0.0.1:3128
```

```
export http_proxy=http://127.0.0.1:3128
```


```
export https_proxy=http://<user - имя пользователя для squid на локальном компьютере>:<пароль для user>@127.0.0.1:3128
```

```
export http_proxy=http://<user - имя пользователя для squid на локальном компьютере>:<пароль для user>@127.0.0.1:3128
```

*Проверяем наш внешний ip*

```
curl ifconfig.me
```

> sudo -E pecl install msgpack
  Флаг -E у sudo важен — он передает ваши переменные окружения (http_proxy) в сессию суперпользователя.
