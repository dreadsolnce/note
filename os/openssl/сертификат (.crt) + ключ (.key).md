***Пример выпуска основного самоподписаного сертификата для web сервера***

> [!NOTE] Описание
> это как создание цифрового удостоверения личности и уникальной секретной подписи для вашего сервера или приложения.


```
openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout tls.key -out tls.crt -subj "/CN=myapp.example.com"
```


> [!NOTE] Информация
> В итоге мы создаем два файла: **ключ** и **сертификат** для имени (доменного имени) myapp.example.com

Эта команда создаст два файла: tls.key (ваш секретный ключ) и tls.crt (ваш публичный сертификат), которые можно сразу использовать, например, в веб-сервере Nginx или Apache для настройки HTTPS.


> [!success] Где использовать
> Такой вариант используется например в ingress кластера kubernetes для настройка шифрованного соединения протокол https
