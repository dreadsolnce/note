## Вспомогательная информация

**CNAME**

> [!NOTE] Title
> Запись CNAME (Canonical Name, или каноническое имя) в DNS нужна для создания псевдонима одного доменного имени для другого.
>   Основные причины использования CNAME-записей:
>   * Создание псевдонимов: Позволяет нескольким доменным именам (например, www.example.com и ftp.example.com) указывать на одно основное (каноническое) доменное имя (example.com).
 >  * Упрощение управления IP-адресами: Если IP-адрес основного домена изменяется, достаточно обновить только A-запись для этого основного домена. Все CNAME-записи, указывающие на него, автоматически будут разрешаться на новый IP-адрес без необходимости их индивидуального изменения.
 >  * Гибкость для различных сервисов: Полезно, когда разные сервисы работают на одном IP-адресе, но требуют разных имен хостов.
 >  * Брендирование и пользовательские ссылки: Используется для создания брендированных или кастомных ссылок, которые перенаправляют трафик на целевой URL, даже если фактическое содержимое размещено на другом домене.
 >
 >Важно помнить, что CNAME-запись всегда должна указывать на другое доменное имя, а не напрямую на IP-адрес. DNS-резолвер, встречая CNAME, продолжает поиск по каноническому имени, чтобы найти соответствующую A-запись (или AAAA-запись) и получить конечный IP-адрес.
 >

## Настройки netplan:

***Путь сохранения настроек для netplan:***

```
ls /etc/netplan/00-installer-config.yaml
```

***Изменение конфигурации ip dns сервера:***

```
sudo systemctl status systemd-resolved.service
```

```
cd /etc/netplan
```

```
sudo cp 00-installer-config.yaml 00-installer-config.yaml.original
```

```
dig @172.16.126.15 dev.border-patrol.ru
```

```
sudo vim 00-installer-config.yaml
```

*Проверить корректность конфигурационного файла Netplan для синтаксической проверки*
```
sudo netplan generate
```

*Для предварительного применения конфигурации и проверки её работоспособности в течение 120 секунд, после чего конфигурация будет отменена или подтверждена*

```
sudo netplan try
```

*или сразу применить новую конфигурацию:*

```
sudo netplan apply
```

*Очистить кэш dns записей:*

```
sudo systemd-resolve --flush-caches
sudo systemctl restart systemd-resolved.service
```

*Проверка применения новых dns серверов:*

```
sudo systemd-resolve --status
```

```
resolvctl status
```

```
nslookup 172.16.3.204
```
