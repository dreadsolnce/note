***Современный способ***

> [!NOTE] Информация
> Если при выполнении обновлении списка пакетов командой `apt-get update` возникает ошибка вида: ***The following signatures were invalid: EXPKEYSIG B188E2B695BD4743 DEB.SURY.ORG Automatic Signing Key <deb@sury.org>***, это означает, что необходимо скачать новый ключ репозитория.
![[Снимок экрана от 2026-04-15 12-10-58.png]]


> [!NOTE] Описание
> Ошибка ***EXPKEYSIG*** означает, что срок действия GPG-ключа, которым подписан репозиторий, истек. Разработчик обновил ключ на сервере, но ваша система все еще использует старый.


- проверяем, что в репозитории находится ключ, для этого откроем в браузере ссылку:

https://mirror.yandex.ru/mirrors/packages.sury.org/php/

![[Снимок экрана от 2026-04-15 12-14-04.png]]

- скачиваем новый ключ репозитория
```
sudo curl -sSLo /usr/share/keyrings/yandex-php.gpg https://mirror.yandex.ru/mirrors/packages.sury.org/php/apt.gpg
```

другая команда

```
curl -fsSL https://packages.gitlab.com/gpgkey/gpg.key | sudo gpg --dearmor -o /usr/share/keyrings/gitlab-runner-archive-keyring.gpg
```

- добавляем новый ключ в файл /etc/apt/sources.list/d/surry-mirror.list

```
sudo vim /etc/apt/sources.list/d/surry-mirror.list
```

```test
+++ deb [signed-by=/usr/share/keyrings/yandex-php.gpg] https://mirror.yandex.ru/mirrors/packages.sury.org/php/ bookworm main
```

![[Снимок экрана от 2026-04-15 12-16-14.png]]

- выполнить повторное обновление списка пакетов

```
sudo apt-get update
```

***Переход от старого способа к новому***


> [!important] Информация
> Если ключ установлен старым способом то можно видеть следующее предупреждение: ***Key is stored in legacy trusted.gpg keyring (/etc/apt/trusted.gpg), see the DEPRECATION section in apt-key(8) for details***


> [!NOTE] Информация
> Это не критическая ошибка, а предупреждение (Warning). Оно означает, что ваша система безопасности apt работает по современным стандартам, а ключ для ClickHouse добавлен «дедовским» способом.
> Минус этого подхода: Если вы добавили ключ для ClickHouse в общую кучу, то этот ключ технически получает право подтверждать подлинность пакетов из любого другого репозитория (например, из официального репозитория Ubuntu). Это дыра в безопасности.
> Современный стандарт требует, чтобы у каждого репозитория был свой отдельный файл ключа, который привязан только к нему.

- список ключей в старом хранилище

```
sudo apt-key list
```

![[Снимок экрана от 2026-04-15 13-43-58.png]]

- Экспорт ключа в отдельный файл (где номер ключа это 8 последних цифр в pub)

```
sudo apt-key export 2B48D754 | sudo gpg --dearmor -o /usr/share/keyrings/clickhouse-keyring.gpg
```

- изменить файл list

```
sudo vim /etc/apt/sources.list/d/clickhouse_com_deb.list
```

```text
+++ deb [signed-by=/usr/share/keyrings/clickhouse-keyring.gpg] https://packages.clickhouse.com/deb stable main
```

![[Снимок экрана от 2026-04-15 13-48-14.png]]

- удалить ключ из старого хранилища

```
sudo apt-key del 2B48D754; sudo apt-key list
```

- выполнить повторное обновление списка пакетов

```
sudo apt-get update
```
